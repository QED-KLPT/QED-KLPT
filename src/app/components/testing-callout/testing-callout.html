Testing Callouts

The following demos showcase the app-callout component with various palettes and configurations following the Queensland Government Design System.

1. Default Callout

A basic callout with the default palette — blue accent border and light blue background. Used for standard important information.

<app-callout
  title="Important Notice"
  message="This is a callout with the default palette..."></app-callout>

2. Light Palette Callout

A callout using the light palette variant with a softer background shade. Useful for less prominent but still important information.

<app-callout
  title="Updated Information"
  message="This callout uses the light palette..."
  variant="light"></app-callout>

3. Alt Palette Callout

The alt palette uses a neutral grey tone. Ideal for supplementary information that does not need strong visual emphasis.

<app-callout
  title="General Information"
  message="The alt palette uses a neutral grey tone..."
  variant="alt"></app-callout>

4. Dark Palette Callout

A callout with a dark blue background and white text. Creates strong visual contrast for critical announcements.

<app-callout
  title="System Maintenance"
  message="This callout uses the dark palette..."
  variant="dark"></app-callout>

5. Dark-alt Palette Callout

The dark-alt variant uses a deeper blue background — the most visually prominent palette option for urgent notifications.

<app-callout
  title="Scheduled Downtime"
  message="The dark-alt variant uses a deeper blue..."
  variant="dark-alt"></app-callout>

6. Screen Reader Only Heading

A callout with a heading that is visually hidden but available to screen readers. The title remains in the markup for assistive technologies.

<app-callout
  title="Accessibility Heading"
  message="This callout has a screen-reader-only heading..."
  srHeading></app-callout>

7. Callout with Links

A callout containing hyperlinks within the message text. Useful for directing users to related resources or documentation.

<app-callout
  title="External Resources"
  message='Callouts can contain links. Visit the <a href="#">QLD Government website</a> for more information.'></app-callout>

8. Multi-paragraph Callout (ng-content)

A callout using <ng-content> to project rich HTML content including multiple paragraphs and formatted text.

 
<app-callout title="Detailed Guidance">
  <p>This callout contains multiple paragraphs...</p>
  <p>The second paragraph expands on the details...</p>
</app-callout>

9. Callout with Checklist

A callout containing a bulleted list via <ng-content>. Ideal for checklists, examples, and case studies.

 
<app-callout title="Assessment Checklist" variant="light">
  <ul>
    <li>Review the assessment criteria</li>
    <li>Gather evidence from observations</li>
  </ul>
</app-callout>

10. Minimal Callout (no message)

A callout with only a title and no message content. Demonstrates the component handling an empty message gracefully.

<app-callout
  title="Minimal Callout"
  message=""></app-callout>
