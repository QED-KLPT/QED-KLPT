Testing Buttons

The following demos showcase the app-qld-button component with various variants, sizes, and configurations following the Queensland Government Design System.

1. Primary Button (Default)

The default primary button is the most prominent action. It uses a solid blue background with white text for main calls-to-action like "Submit" or "Save".

Primary Button 
<app-qld-button variant="primary">Primary Button</app-qld-button>

2. Secondary Button

The secondary button uses an outline style with a colored border. It provides an alternative action that is less prominent than the primary button, such as "Cancel" or "Back".

Secondary Button 
<app-qld-button variant="secondary">Secondary Button</app-qld-button>

3. Tertiary Button (Underlined)

The tertiary button is the least visually prominent option. It appears as underlined text, used for low-emphasis actions like "Learn more" or "Skip".

Tertiary Button 
<app-qld-button variant="tertiary">Tertiary Button</app-qld-button>

4. Primary with Leading Icon

A primary button with an icon placed before the text. Useful for actions like "Save" or "Upload" where a visual cue reinforces the action.

Save 
<app-qld-button variant="primary" iconLead="true">
  <i class="fa-light fa-save" aria-hidden="true"></i>Save
</app-qld-button>

5. Primary with Trailing Icon

A primary button with an icon placed after the text. Commonly used for navigation actions like "Next" or "View details".

Next 
<app-qld-button variant="primary" iconTrail="true">
  Next<i class="fa-light fa-arrow-right" aria-hidden="true"></i>
</app-qld-button>

6. Secondary with Leading Icon

A secondary (outlined) button with a leading icon. Combines the alternative action style with visual context for clarity.

Download 
<app-qld-button variant="secondary" iconLead="true">
  <i class="fa-light fa-download" aria-hidden="true"></i>Download
</app-qld-button>

7. Disabled Primary Button

A disabled primary button with reduced opacity and a not-allowed cursor. Used when an action is temporarily unavailable, such as before form validation passes.

Disabled Primary 
<app-qld-button variant="primary" [disabled]="true">Disabled Primary</app-qld-button>

8. Disabled Tertiary Button

A disabled tertiary button in its underlined text style. The underline is removed when disabled to indicate the action is unavailable.

Disabled Tertiary 
<app-qld-button variant="tertiary" [disabled]="true">Disabled Tertiary</app-qld-button>

9. Large Primary Button

A larger primary button with increased height and font size. Used for prominent actions that need extra visual weight, such as a main form submission.

Large Primary Button 
<app-qld-button variant="primary" size="lg">Large Primary Button</app-qld-button>

10. Small Secondary Button

A compact secondary button with reduced height and font size. Ideal for dense layouts or when space is limited, such as in toolbars or data tables.

Small Secondary 
<app-qld-button variant="secondary" size="sm">Small Secondary</app-qld-button>
