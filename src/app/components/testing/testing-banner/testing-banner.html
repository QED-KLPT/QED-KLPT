Testing Banners

The following demos showcase the app-banner component with various variants, backgrounds, and configurations following the Queensland Government Design System.

1. Default White Banner (No Breadcrumbs)

A minimal banner with just a title on a white background. Used for simple page headers without navigation context.

<app-banner variant="default" title="Default White Banner"></app-banner>

2. Default Banner with Breadcrumbs

A default banner with breadcrumb navigation showing the page hierarchy. The standard pattern for content pages.

<app-banner variant="default" title="Default Banner with Breadcrumbs"
  [breadcrumbs]="defaultBreadcrumbs"></app-banner>

3. Light Background Banner

A default banner with a light grey background for subtle visual distinction. Useful when you want to separate the header from page content.

<app-banner variant="default" title="Light Background Banner"
  background="light"
  [breadcrumbs]="defaultBreadcrumbs"></app-banner>

4. Alternate Background Banner

A default banner with an alternate background color variant. Provides more visual weight than the light variant.

<app-banner variant="default" title="Alternate Background Banner"
  background="alt"
  [breadcrumbs]="defaultBreadcrumbs"></app-banner>

5. Dark Background Banner

A default banner with a dark background and white text. Creates strong visual contrast for important page headers.

<app-banner variant="default" title="Dark Background Banner"
  background="dark"
  [breadcrumbs]="darkBreadcrumbs"></app-banner>

6. Basic Banner with Abstract

A basic banner variant that includes a primary heading and an abstract/description paragraph. Ideal for pages needing brief context.

<app-banner variant="basic" title="Basic Banner Example"
  [breadcrumbs]="defaultBreadcrumbs"
  [abstractText]="basicAbstract"></app-banner>

7. Basic Banner with Background Image

A basic banner with a background image overlay. The abstract text sits on top of the image for visual impact.

<app-banner variant="basic" title="Basic Banner with Image"
  background="dark"
  [breadcrumbs]="defaultBreadcrumbs"
  [abstractText]="basicAbstract"></app-banner>

8. Advanced Banner with Hero Image

An advanced banner with a hero image on the left and content on the right. The two-column layout is responsive, stacking vertically on mobile.

<app-banner variant="advanced" title="Advanced Banner with Hero"
  [breadcrumbs]="defaultBreadcrumbs"
  [abstractText]="advancedAbstract"
  [heroImageUrl]="heroImageUrl"
  [heroImageAlt]="heroImageAlt"></app-banner>

9. Advanced Banner with CTAs

An advanced banner with primary and secondary call-to-action buttons. Used when you need to guide users toward specific actions.

<app-banner variant="advanced" title="Advanced Banner with CTAs"
  [breadcrumbs]="defaultBreadcrumbs"
  [abstractText]="advancedAbstract"
  [heroImageUrl]="heroImageUrl"
  [heroImageAlt]="heroImageAlt"
  [ctaPrimary]="ctaPrimary"
  [ctaSecondary]="ctaSecondary"></app-banner>

10. Advanced Banner with Icon Tiles

An advanced banner with icon tile navigation. Each tile is a clickable link with an icon, ideal for quick access to key sections or contact methods.

<app-banner variant="advanced" title="Advanced Banner with Icon Tiles"
  [breadcrumbs]="defaultBreadcrumbs"
  [abstractText]="advancedAbstract"
  [heroImageUrl]="heroImageUrl"
  [heroImageAlt]="heroImageAlt"
  [ctaPrimary]="ctaPrimary"
  [iconTiles]="iconTiles"></app-banner>
