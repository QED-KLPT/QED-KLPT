Testing Radio

The following demos showcase the app-radio-group component with various states, sizes, and layouts following the Queensland Government Design System.

1. Default Vertical Radio Buttons

The standard radio button group with a required legend, hint text, and vertically stacked options. This is the most common pattern for form selection.

<app-radio-group label="Select your device"
  [options]="defaultOptions"
  hint="Select an option"
  required></app-radio-group>

2. Small Radio Buttons

Smaller radio buttons that are less visually prominent. Useful for compact layouts like search result filters where you want to minimize visual weight.

<app-radio-group label="Filter by device"
  [options]="smallOptions"
  hint="Select an option"
  size="small"></app-radio-group>

3. Inline/Horizontal Layout

Radio buttons displayed horizontally in a row. Best for simple binary or ternary choices like yes/no/unsure where options are short.

<app-radio-group label="Do you agree?"
  [options]="inlineOptions"
  hint="Choose one"
  inline></app-radio-group>

4. Valid State

A radio group with a valid/confirmed state indicated by a green success message and border. Used when the user has made an acceptable selection.

<app-radio-group label="Select your device"
  [options]="validOptions"
  successMessage="Selection confirmed"></app-radio-group>

5. Error State

A radio group with an error state indicated by a red error message and bordered inputs. Used when the user must make a selection before proceeding.

<app-radio-group label="Select your device"
  [options]="errorOptions"
  hint="Select an option"
  error="Please select a device"></app-radio-group>

6. Disabled Radio Buttons

All radio buttons in the group are disabled, indicating they are no longer available for selection. Useful for expired or invalid options.

<app-radio-group label="Select your device"
  [options]="disabledOptions"
  hint="This option is no longer available"
  disabled></app-radio-group>

7. Optional Label

A radio group where the selection is not required, indicated by an "(Optional)" tag next to the legend. Users can choose to skip this question.

<app-radio-group label="Preferred contact method"
  [options]="optionalOptions"
  hint="Choose how we can reach you"
  optional></app-radio-group>

8. Mixed Enabled/Disabled Options

A radio group where some individual options are disabled while others remain selectable. Useful for showing unavailable choices alongside active ones.

<app-radio-group label="Select your device"
  [options]="mixedOptions"
  hint="Some options may be unavailable"></app-radio-group>

9. Required Without Hint

A required radio group marked with an asterisk but without hint text. Used when the question is self-explanatory and no additional guidance is needed.

<app-radio-group label="Select an option"
  [options]="requiredNoHintOptions"
  required></app-radio-group>

10. Minimal Radio Buttons

A simple radio group with no legend, hint, or required marker. Used for compact inline selections where context is already clear from surrounding content.

<app-radio-group label="Your opinion"
  [options]="minimalOptions"></app-radio-group>
