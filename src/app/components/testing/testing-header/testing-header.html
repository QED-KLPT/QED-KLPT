Testing Header

The following demos showcase the <klpt-header> component with various configurations following the Queensland Government Design System patterns.

1. Full Header (Pre-header + Coat of Arms + Search + Navigation)

The complete header with dark pre-header bar showing qld.gov.au, Coat of Arms branding, search functionality, and primary navigation. This is the standard pattern for most Queensland Government websites.

{{ example1Code }}

2. Header Without Pre-header (Masthead + Navigation)

A simplified header without the dark pre-header utility bar. Used when the qld.gov.au government branding is not required.

{{ example2Code }}

3. Header with Custom Logo

Uses a custom logo image instead of the Queensland Coat of Arms. Suitable for agencies that have their own approved branding.

{{ example3Code }}

4. Header with Site Name Only

A text-only header without any branding image. Used for internal tools or applications where government branding is not applicable.

{{ example4Code }}

5. Compact Header (Pre-header + Brand Only)

A minimal header with just the pre-header and brand. No search or navigation - used for simple pages that don't require site-wide navigation.

{{ example5Code }}

6. Header with Search but No Navigation

A header that includes the search functionality but omits the primary navigation bar. Useful for content-focused pages.

{{ example6Code }}

7. Minimal Header (Brand + Hamburger Menu)

A mobile-first minimal header showing only the brand and hamburger menu toggle. Navigation appears as a slide-out drawer on mobile/tablet.

{{ example7Code }}

8. Header with Tagline

A header that includes a tagline/subheading beneath the site name. Used when additional context about the site is helpful.

{{ example8Code }}

9. Header with Secondary Navigation Bar

A header with an additional secondary navigation bar below the primary nav. Used for sites with multiple content sections or user roles.

{{ example9Code }}

10. Contained Header (Narrower Width)

A header variant that is contained within a narrower width rather than full-bleed. Used for applications where content doesn't span the full viewport width.
