Testing Navigation

The following demos showcase various navigation patterns from the Queensland Government Design System, including horizontal main navigation, vertical side navigation, and in-page navigation components.

1. Simple Horizontal Navigation (Light)

A basic horizontal navigation bar with simple text links and light theme. The active item is highlighted to indicate the current page. This is the standard pattern for primary site navigation with straightforward information architecture.

<app-qld-main-nav
  [items]="horizontalLightItems"
  theme="light"
  activeItem="Learning domains">
</app-qld-main-nav>

2. Dark Horizontal Navigation

A dark-themed horizontal navigation bar using the Queensland Government Design System dark variant. Suitable for sites that use a dark color scheme or want higher contrast navigation with better visibility.

<app-qld-main-nav
  [items]="horizontalDarkItems"
  theme="dark"
  activeItem="KLPT">
</app-qld-main-nav>

3. Horizontal Navigation with Icons

Horizontal navigation with Font Awesome icons preceding each link label. Icons help users quickly identify menu items and reduce cognitive load. Best used sparingly for the most important navigation categories.

<app-qld-main-nav
  [items]="horizontalIconItems"
  theme="light"
  activeItem="Learning domains">
</app-qld-main-nav>

4. Horizontal Navigation with Dropdowns

A navigation item with a dropdown submenu containing child links. The submenu appears on hover for desktop and on click for mobile. Ideal for sites with a two-level information hierarchy that need to organize related pages.

<app-qld-main-nav
  [items]="horizontalDropdownItems"
  theme="light"
  activeItem="Foundations">
</app-qld-main-nav>

5. Mega Menu with Descriptions

A mega menu where each sub-link includes a short description paragraph beneath it. This provides additional context to help users understand what each link leads to before clicking, improving content findability and user experience.

<app-qld-main-nav
  [items]="horizontalMegaMenuItems"
  theme="light"
  activeItem="Foundations">
</app-qld-main-nav>

6. Basic Vertical Navigation

A simple flat list of navigation links displayed vertically. Used for side navigation panels that help users move between related pages within the same section or topic area of a website.

<app-vertical-nav
  [items]="verticalBasicItems">
</app-vertical-nav>

7. Vertical Navigation with Accordion Submenus

Expandable and collapsible nested navigation sections. Click the chevron button to toggle submenu visibility. Uses aria-expanded for accessibility and helps organize large amounts of navigation content hierarchically.

<app-vertical-nav
  [items]="verticalAccordionItems">
</app-vertical-nav>

8. Vertical Navigation with Icons and Active State

Navigation links with Font Awesome icons alongside labels, combined with an active page state highlighted by a left border accent and purple background. The active item receives aria-current="page" styling for accessibility.

<app-vertical-nav
  [items]="verticalIconActiveItems">
</app-vertical-nav>

9. Default In-Page Navigation

A basic in-page navigation with the default "On this page" heading and a standard set of links. Used as a table of contents on lengthy pages to provide structural overview and quick access to different sections.

<app-in-page-nav
  [items]="inPageBasicItems">
</app-in-page-nav>

10. In-Page Navigation with Custom Heading and Dark Variant

The heading can be customized to match the context of your page. The dark variant uses lighter border color and link styling suitable for placement on dark backgrounds, ensuring sufficient contrast and visual harmony.

<div style="background: var(--doe-color-heading);">
  <app-in-page-nav
    heading="Contents"
    [items]="inPageCustomDarkItems"
    variant="dark">
  </app-in-page-nav>
</div>
