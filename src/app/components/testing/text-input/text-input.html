Testing Text Input

The following demos showcase the app-text-input component with various input types, states, and configurations following the Queensland Government Design System.

1. Basic Text Input

A simple text input with a label only, no additional options.

<app-text-input label="Full name"></app-text-input>

2. Email Input

An email-type input with placeholder text to guide the user on the expected format.

<app-text-input label="Email address"
  placeholder="e.g. john.smith@example.com"
  type="email"></app-text-input>

3. Telephone Input

A telephone-type input with placeholder text showing the expected phone number format.

<app-text-input label="Phone number"
  placeholder="e.g. 04XX XXX XXX"
  type="tel"></app-text-input>

4. Password Input

A password-type input that masks the entered characters for security.

<app-text-input label="Password"
  placeholder="Enter your password"
  type="password"></app-text-input>

5. URL Input

A URL-type input with placeholder text showing the expected web address format.

<app-text-input label="Website"
  placeholder="https://example.com"
  type="url"></app-text-input>

6. Number Input

A number-type input that restricts input to numeric values only.

<app-text-input label="Age"
  placeholder="Enter your age"
  type="number"></app-text-input>

7. With Hint Text

A text input with hint text below the label, linked via aria-describedby for accessibility.

<app-text-input label="Hint Text Example"
  hint="Enter your mobile number with the area code included."></app-text-input>

8. Error State

A text input with an error message displayed in red, using the qld__text-input--error class.

<app-text-input label="Error State"
  placeholder="Enter your address"
  error="Please provide a valid street address."></app-text-input>

9. Success/Valid State

A text input with a success message displayed in green, using the qld__text-input--valid class.

<app-text-input label="Success State"
  placeholder="Enter your username"
  success="This username is available."></app-text-input>

10. Disabled State

A non-interactive disabled text input that cannot be edited.

<app-text-input label="Disabled Field"
  placeholder="This field is disabled"
  [disabled]="true"></app-text-input>
