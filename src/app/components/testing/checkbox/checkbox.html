Testing Checkbox

The following demos showcase the app-checkbox component with various attributes and configurations.

1. Basic Checkbox

A simple checkbox with a label only, no additional options.

<app-checkbox label="Agree to terms"></app-checkbox>

2. Checked State

A checkbox that is pre-checked by default.

<app-checkbox label="Subscribe to newsletter" [checked]="true"></app-checkbox>

3. Required Field

A checkbox with a required indicator (asterisk) and aria-required attribute.

<app-checkbox label="I confirm the information is correct" [required]="true"></app-checkbox>

4. Optional Field

A checkbox with an "(Optional)" indicator in the label.

<app-checkbox label="Receive marketing materials" [optional]="true"></app-checkbox>

5. With Hint Text

A checkbox with hint text below the label, linked via aria-describedby.

<app-checkbox label="Enable notifications"
  hint="You can change this setting at any time in your account preferences."></app-checkbox>

6. Error State

A checkbox with an error message displayed in red, using the qld__checkbox--error class.

<app-checkbox label="Accept privacy policy"
  error="You must accept the privacy policy to continue."></app-checkbox>

7. Success/Valid State

A checked checkbox indicating a verified or completed state.

<app-checkbox label="Terms verified" [checked]="true"></app-checkbox>

8. Disabled State

A non-interactive disabled checkbox that cannot be toggled.

<app-checkbox label="Disable this option" [disabled]="true"></app-checkbox>

9. Indeterminate State

A checkbox in a partial/indeterminate state, often used for "select all" parent checkboxes.

<app-checkbox label="Select all options" [indeterminate]="true"></app-checkbox>

10. Complex Form

A checkbox combining multiple features: required label, hint text, and error message.

<app-checkbox label="Consent to data processing"
  [required]="true"
  hint="Your data will be processed in accordance with the GDPR regulations."
  error="Consent is required to proceed with your application."></app-checkbox>
