Testing Text Area

The following demos showcase the app-textarea component with various attributes and configurations.

1. Basic Text Area

A simple textarea with a label only, no additional options.

<app-textarea label="Message"></app-textarea>

2. With Placeholder

A textarea with placeholder text shown inside the field to guide the user.

<app-textarea label="Message" placeholder="Enter your message here..."></app-textarea>

3. Required Field

A textarea with a required indicator (asterisk) and aria-required attribute.

<app-textarea label="Feedback" [required]="true"></app-textarea>

4. Optional Field

A textarea with an "(Optional)" indicator in the label.

<app-textarea label="Comments" [optional]="true"></app-textarea>

5. With Hint Text

A textarea with hint text below the label, linked via aria-describedby.

<app-textarea label="Additional Information"
  hint="Provide any extra details that might help us understand your request better."></app-textarea>

6. Error State

A textarea with an error message displayed in red, using the qld__text-input--error class.

<app-textarea label="Problem Description"
  error="Please describe the problem you are experiencing."></app-textarea>

7. Success/Valid State

A textarea with a success message displayed in green, using the qld__text-input--valid class.

<app-textarea label="Resolution Notes"
  success="Your notes have been saved successfully."></app-textarea>

8. Disabled State

A non-interactive disabled textarea that cannot be edited.

<app-textarea label="Notes" [disabled]="true"></app-textarea>

9. Block Variant

A full-width textarea that expands to fill the available width using qld__text-input--block.

<app-textarea label="Full-width Message" [block]="true"></app-textarea>

10. Complex Form

A textarea combining multiple features: required label, hint text, and error message.

<app-textarea label="Detailed Feedback"
  hint="Include as much detail as possible to help us address your concerns."
  error="A detailed description is required for us to investigate this issue."></app-textarea>
