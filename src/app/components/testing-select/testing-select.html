Testing Select

The following demos showcase the app-select-box component with various states, sizes, and configurations following the Queensland Government Design System.

1. Default Select

A basic select dropdown with a label, hint text, and predefined options. The default pattern for single-choice form fields.

<app-select-box label="Select an option"
  [options]="defaultOptions"
  hint="Choose from the list below"></app-select-box>

2. Filled Style Select

A select with a filled background style, achieved by wrapping in the qld__form-style-filled container class. Provides a visually distinct input style used site-wide.

<div class="qld__form-style-filled">
  <app-select-box label="Select an option"
    [options]="defaultOptions"
    hint="Filled style variant"></app-select-box>
</div>

3. Block Width Select

A select that spans the full width of its container using the block input. Useful for forms where consistent full-width alignment is desired.

<app-select-box label="Select an option"
  [options]="defaultOptions"
  block></app-select-box>

4. Half-Width Field

A select constrained to half the container width using the qld__field-width--half class. Ideal for multi-column form layouts where fields sit side by side.

Half-width field 
<select class="qld__select-control qld__field-width--half
  qld__text-input--block">
  <option value="" disabled selected>Please select</option>
  <option value="1">Option one</option>
  <option value="2">Option two</option>
  <option value="3">Option three</option>
</select>

5. Disabled Select

A disabled select dropdown that cannot be interacted with. Used when the field is temporarily unavailable or has been completed elsewhere.

<app-select-box label="Select an option"
  [options]="disabledOptions"
  disabled></app-select-box>

6. Required Select

A required select field marked with a red asterisk and aria-required="true". The user must make a selection before the form can be submitted.

<app-select-box label="Select an option"
  [options]="requiredOptions"
  required></app-select-box>

7. Optional Select

A select field marked as optional with an "(Optional)" tag in the label. Users can skip this selection without error.

<app-select-box label="Select an option"
  [options]="optionalOptions"
  optional></app-select-box>

8. Error State

A select in an error state with a red border and error message. Used when the user has not made a valid selection or submitted an invalid choice.

<app-select-box label="Select a category"
  [options]="errorOptions"
  error="Please select a category"></app-select-box>

9. Success/Valid State

A select in a valid/success state with a green border and success message. Indicates the user has made an acceptable selection.

<app-select-box label="Select a region"
  [options]="successOptions"
  success="Selection confirmed"></app-select-box>

10. Complex Form with Multiple Selects

A multi-field form demonstrating multiple select boxes in a realistic assessment context. Shows how selects work together in complex forms with different labels, hints, and options.

<div style="display: grid; gap: 1.5rem;">
  <app-select-box label="Learning domain"
    [options]="domainOptions"
    hint="Select the relevant learning domain"></app-select-box>
  <app-select-box label="Assessment type"
    [options]="assessmentOptions"
    hint="Choose the type of assessment"></app-select-box>
  <app-select-box label="Status"
    [options]="statusOptions"
    hint="Current status of this assessment"></app-select-box>
</div>
