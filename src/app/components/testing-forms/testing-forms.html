Testing Forms

The following demos showcase form components based on the Queensland Government Design System, including text inputs, radio buttons, select boxes, and fieldsets.

1. Basic Text Input

A simple text input field with a label only, no additional options or constraints.

<div class="qld__form-group">
  <label for="email" class="qld__label">Email address</label>
  <input type="email" id="email" class="qld__text-input qld__field-width--half">
</div>

2. Text Input with Hint Text

A text input with hint text below the label, providing additional guidance to users. Linked via aria-describedby for accessibility.

Tax file numbers are 9 digits 
<div class="qld__form-group">
  <label for="tfn" class="qld__label">What is your tax file number (TFN)?</label>
  <span class="qld__hint-text" id="tfn-hint">Tax file numbers are 9 digits</span>
  <input type="text" id="tfn" class="qld__text-input qld__field-width--md qld__text-input--number"
    inputmode="numeric" aria-describedby="tfn-hint">
</div>

3. Required Field

A text input with a required indicator (red asterisk) and aria-required attribute. Used when user input is mandatory.

<div class="qld__form-group">
  <label for="name" class="qld__label qld__label--required">Full name</label>
  <input type="text" id="name" class="qld__text-input qld__field-width--half" aria-required="true">
</div>

4. Optional Field

A text input with an "(Optional)" indicator in the label, clearly communicating to users that this field is not mandatory.

<div class="qld__form-group">
  <label for="phone" class="qld__label">Phone number<span class="qld__label--optional"> (Optional)</span></label>
  <input type="tel" id="phone" class="qld__text-input qld__field-width--half">
</div>

5. Error State

A text input with an error message displayed in red, using the qld__text-input--error class. Provides immediate feedback to users.

Please enter a valid email address. 
<div class="qld__form-group">
  <label for="email" class="qld__label">Email address</label>
  <span class="qld__input--error" id="email-error">
    <svg class="qld__icon qld__icon--lead qld__icon--sm" role="img" aria-label="Error: ">
      <use href="./?a=169317:dist/mysource_files/img/QLD-icons.svg#status-error"></use>
    </svg>
    Please enter a valid email address.
  </span>
  <input type="email" id="email" class="qld__text-input qld__field-width--half qld__text-input--error"
    aria-describedby="email-error" aria-invalid="true">
</div>

6. Success/Valid State

A text input with a success message displayed in green, using the qld__text-input--valid class to indicate valid input.

This email address is available. 
<div class="qld__form-group">
  <label for="email" class="qld__label">Email address</label>
  <span class="qld__input--success" id="email-success">This email address is available.</span>
  <input type="email" id="email" class="qld__text-input qld__field-width--half qld__text-input--valid"
    aria-describedby="email-success">
</div>

7. Radio Button Group

A group of radio buttons within a fieldset, allowing users to select one option from multiple choices. Uses qld__fieldset and qld__fieldset__legend.

<app-radio-group
  label="Do you have a Queensland driver's licence?"
  name="licence"
  [options]="[
    {{ '{' }} label: 'Yes', value: 'yes' {{ '}' }},
    {{ '{' }} label: 'No', value: 'no' {{ '}' }}
  ]"></app-radio-group>

8. Select Box / Dropdown

A select box presenting multiple options in a constrained space. Useful when space is limited or there are many options.

<app-select-box
  label="Choose an option"
  name="demo8"
  [options]="[
    {{ '{' }} label: 'Choose an option...', value: '' {{ '}' }},
    {{ '{' }} label: 'Option one', value: 'option1' {{ '}' }},
    {{ '{' }} label: 'Option two', value: 'option2' {{ '}' }}
  ]"></app-select-box>

9. Fieldset with Grouped Fields

A fieldset grouping related form fields together, with a legend providing context. Commonly used for address blocks or personal information sections.

Your address 
<fieldset class="qld__fieldset">
  <legend class="qld__fieldset__legend">Your address</legend>
  <div class="qld__form-group">
    <label for="line1" class="qld__label qld__label--required">Street address</label>
    <input type="text" id="line1" class="qld__text-input qld__field-width--half" aria-required="true">
  </div>
  <div class="qld__form-group">
    <label for="line2" class="qld__label">Unit or suite number<span class="qld__label--optional"> (Optional)</span></label>
    <input type="text" id="line2" class="qld__text-input qld__field-width--half">
  </div>
  <div class="qld__form-group">
    <label for="suburb" class="qld__label qld__label--required">Suburb or locality</label>
    <input type="text" id="suburb" class="qld__text-input qld__field-width--half" aria-required="true">
  </div>
  <div class="qld__form-group">
    <label for="postcode" class="qld__label qld__label--required">Postcode</label>
    <input type="text" id="postcode" class="qld__text-input qld__field-width--half qld__text-input--number"
      inputmode="numeric" aria-required="true">
  </div>
</fieldset>

10. Complete Form Example

A complete form combining multiple elements: radio buttons for gender selection, select boxes for learning domain and assessment type, a text input for notes, and a checkbox for consent.

Child assessment form
Gender 
<fieldset class="qld__fieldset">
  <legend class="qld__fieldset__legend">Child assessment form</legend>
  <app-radio-group name="gender" [options]="genderOptions"></app-radio-group>
  <app-select-box label="Learning domain" [options]="domainOptions"></app-select-box>
  <app-select-box label="Assessment type" [options]="assessmentOptions"></app-select-box>
  <app-textarea label="Assessment notes" hint="Provide additional observations." rows="4"></app-textarea>
  <app-checkbox label="I confirm this assessment is accurate" required></app-checkbox>
</fieldset>
