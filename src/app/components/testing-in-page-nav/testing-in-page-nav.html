Testing In-Page Navigation

The following demos showcase the app-in-page-nav component with various configurations following the Queensland Government Design System.

1. Default In-Page Navigation

A basic in-page navigation with the default "On this page" heading and a standard set of links. Used as a table of contents on lengthy pages to provide structural overview.

<app-in-page-nav [items]="basicLinks"></app-in-page-nav>

2. Custom Heading

The heading can be customized to match the context of your page. Common alternatives include "Contents", "Table of Contents", or "This Section".

<app-in-page-nav heading="Contents" [items]="customHeadingLinks"></app-in-page-nav>

3. Short Navigation (3 Links)

In-page navigation works well with any number of links. For shorter pages with only a few sections, a minimal nav keeps the interface clean.

<app-in-page-nav [items]="shortLinks"></app-in-page-nav>

4. Long Navigation (10+ Links)

For very long pages with many sections, the in-page navigation provides a quick way for users to jump to any part of the content. Consider using this when pages exceed 2000 words.

<app-in-page-nav [items]="longLinks"></app-in-page-nav>

5. Dark Background Variant

The dark variant uses a lighter border color and link styling suitable for placement on dark backgrounds. This ensures sufficient contrast and visual harmony.

<div style="background: dark;">
  <app-in-page-nav [items]="links" variant="dark"></app-in-page-nav>
</div>

6. Section-Specific Navigation (H2 Only)

When content is organized primarily into H2 sections without significant subsections, include only H2 headings in the navigation to keep it simple and scannable.

<app-in-page-nav [items]="h2OnlyLinks"></app-in-page-nav>

7. Navigation with Descriptive Links

Link text should clearly describe the section it navigates to. Longer, more descriptive labels help users understand what each section contains before clicking.

<app-in-page-nav [items]="descriptiveLinks"></app-in-page-nav>

8. Nested Section Navigation

For pages with both H2 and H3 sections, include all relevant heading levels. This provides a complete structural overview of the page content hierarchy.

<app-in-page-nav [items]="nestedLinks"></app-in-page-nav>

9. Sticky Positioning

For very long pages, the in-page navigation can be made sticky so it remains visible as users scroll. This is achieved by wrapping the component in a container with position: sticky.

<div style="max-height: 200px; overflow-y: auto;">
  <app-in-page-nav [items]="stickyLinks"></app-in-page-nav>
</div>

10. Accessibility-Focused Navigation

This example demonstrates proper ARIA attributes. The component automatically wraps content in a <nav> element with an aria-label attribute for assistive technology discovery. Each link text clearly describes its destination section.

<app-in-page-nav [items]="accessibilityLinks" heading="On this page"></app-in-page-nav>
