Testing File Upload

The following demos showcase the app-file-upload component with various configurations and states based on the Queensland Government Design System file upload patterns.

1. Basic Upload Field

A simple file upload field with a label, hint text, and select button. Single file upload with no constraints specified.

<app-file-upload
  label="Upload your document"
  hint="Select a file from your device to upload."
  (filesSelected)="onExample1Files($event)"
  (fileRemoved)="onExample1Removed($event)"></app-file-upload>

2. Drag and Drop

A file upload with drag-and-drop support and file type constraints. Users can either drag files onto the drop zone or click to select files.

<app-file-upload
  label="Upload images"
  hint="Drag and drop your image files here or use the button below."
  [description]="example2Description"
  accept=".jpg,.jpeg,.png,.gif"
  [maxSize]="5"
  (filesSelected)="onExample2Files($event)"
  (fileRemoved)="onExample2Removed($event)"></app-file-upload>

3. Multiple Files

A file upload that allows selecting multiple files at once. Users can upload up to 5 files in a single selection.

<app-file-upload
  label="Upload project files"
  hint="You can select multiple files at once."
  [description]="example3Description"
  accept=".docx,.pdf,.xlsx"
  [maxFiles]="5"
  [multiple]="true"
  (filesSelected)="onExample3Files($event)"
  (fileRemoved)="onExample3Removed($event)"></app-file-upload>

4. With File Size Limit

A file upload with a strict maximum file size constraint. Files exceeding the limit will show an error state.

<app-file-upload
  label="Upload report"
  hint="Reports must be under 10MB in size."
  [description]="example4Description"
  accept=".pdf,.docx"
  [maxSize]="10"
  (filesSelected)="onExample4Files($event)"
  (fileRemoved)="onExample4Removed($event)"></app-file-upload>

5. Required Field

A required file upload field indicated by an asterisk. This is used when a file upload is mandatory for form submission.

<app-file-upload
  label="Upload supporting documents *"
  hint="Please upload all required supporting documentation."
  [description]="example5Description"
  accept=".pdf,.jpg,.png"
  [maxSize]="5"
  [required]="true"
  (filesSelected)="onExample5Files($event)"
  (fileRemoved)="onExample5Removed($event)"></app-file-upload>

6. Disabled State

A disabled file upload field where users cannot interact with it. Used when the upload is not yet available or has been restricted.

<app-file-upload
  label="Upload evidence files"
  hint="This upload is currently disabled."
  [description]="example6Description"
  accept=".mp4,.mov"
  [maxSize]="50"
  [disabled]="true"></app-file-upload>

7. Uploading (Loading Spinner)

A file upload in the uploading state. The dropzone is disabled and a loading spinner replaces the upload icon to indicate processing.

<app-file-upload
  label="Upload batch files"
  hint="Files are being processed. Please wait."
  [description]="example7Description"
  accept=".csv,.xlsx"
  [maxSize]="20"
  [uploading]="true"></app-file-upload>

8. Error State

A file upload showing error states with retry and remove options. Errors can occur due to network issues, invalid file types, or size limits.

<app-file-upload
  label="Upload compliance documents"
  hint="Some files failed to upload. Please retry or remove them."
  [description]="example8Description"
  accept=".pdf,.docx"
  [maxSize]="15"
  (filesSelected)="onExample8Files($event)"
  (fileRemoved)="onExample8Removed($event)"></app-file-upload>

A dark-themed file upload component for use on dark backgrounds or in dark mode interfaces.

A file upload demonstrating the full lifecycle: selecting files, loading state, complete status, and successful uploads with file count summary.
